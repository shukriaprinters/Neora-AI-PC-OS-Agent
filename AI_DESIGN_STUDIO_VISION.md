# NEORA DESIGN OS V2 - VOLUME 1, CHAPTER 1, PART 4.2.1: AI DESIGN STUDIO
## Vision & Responsibilities (Version 5.0)

This master specification defines the permanent design vision, core responsibilities, operational boundaries, and integration pipelines for the **AI Design Studio** module within Neora Design OS.

---

## 1. Product Positioning

The AI Design Studio is **NOT**:
*   *A simple image generator:* It does not spit out static, un-editable raster graphics.
*   *A prompt template helper:* It does not merely wrap user text in pre-baked prompt chains.
*   *A basic graphic editor:* It does not rely on the user to carry out all mechanical operations manually.

Instead, the AI Design Studio represents the **Creative Intelligence Center** of Neora Design OS. It is a highly-capable design director, capable of reasoning, planning, styling, and reviewing visual compositions before they are translated onto the active workspace artboard.

---

## 2. Core Responsibilities

```
+-------------------------------------------------------------+
|                     AI DESIGN STUDIO                        |
+-------------------------------------------------------------+
|  1. INTENT REASONING    ---> Analyzes audience & format     |
|  2. STRATEGIC PLANNING  ---> Lays out grid, bleed, & margins |
|  3. ARTISTIC SYNTHESIS  ---> Composes fonts & color harmony  |
|  4. CRITICAL AUDITING   ---> Evaluates contrast & collisions |
+-------------------------------------------------------------+
```

To deliver enterprise-grade visual output, the Design Studio is programmatically structured around four fundamental pillars:

### I. Intent Reasoning & Context Awareness
*   **Target Audience Analysis:** Determines the visual language appropriate for the task (e.g., corporate gold-and-black luxury vs. bright cultural festival tones).
*   **Media and Format Constraints:** Evaluates layout sizing requirements based on target destinations (e.g., printable flyers with margin bleed vs. digital social covers with safe centers).
*   **Multilingual Script Shaping:** Programmatically handles Bangla, English, and Arabic typography with appropriate script constraints to avoid glyph clipping.

### II. Composition & Visual Planning
*   **Grid Blueprint Compilation:** Lays out percentage-based margins, safe guides, and horizontal alignments.
*   **Visual Saliency Management:** Programmatically identifies high-importance focal points, ensuring titles and critical brand assets are positioned with clean negative space.

### III. Aesthetic & Color Synthesis
*   **Mathematical Color Harmonies:** Selects color swatches using proven, accessible visual relationships (Monochromatic, Complementary, Triadic).
*   **Typographic Hierarchy:** Pairs font weights and styles that reflect the requested mood while maintaining maximum legibility.

### IV. Quality Auditing & Review
*   **Collision Auditing:** Automatically checks text layers for overlapping bounds.
*   **Contrast Diagnostics:** Computes real-time background-to-text contrast ratios to preserve WCAG AA legibility levels.

---

## 3. Human-AI Collaboration Framework

```
                 AI ASSISTS  <===>  HUMAN DECIDES
```

The AI Design Studio respects the creative sovereignty of the human designer. The interaction model is guided by clear boundaries of execution:

*   **No Unsolicited Alterations:** The AI must never overwrite lock states, reset custom text contents, or alter user-edited layer dimensions without conversational request.
*   **Interactive Preserves:** Every element generated by the AI is returned as a fully interactive layer. There are no flat lock-ins. The user can drag, scale, and recolor any individual component.
*   **Reasoning Logs Panel:** Every design alteration is accompanied by a transparent log entry in the user-visible system logs, explaining the stylistic justification (e.g., *"Shifted paragraph box down to clear main logo bleed"*).

---

## 4. Multi-Modal Reference Translation

When a reference design or asset is uploaded, the Design Studio avoids copying copyrighted art. Instead, it processes references through high-level feature extraction:

1.  **Palette Extraction:** Identifies dominant, secondary, and accent colors.
2.  **Typographic Classification:** Determines whether the styling uses heavy display sans-serif fonts, classic serif fonts, or script-heavy calligraphic glyphs.
3.  **Layout Hierarchy Mapping:** Traces the spatial coordinates of focal points and groups to replicate the "flow" of the original design without copying the exact visual content.

---

## 5. System Integration & Handoff Rules

The AI Design Studio functions as a unified coordinator across the various subsystems:
*   **To State Managers:** Transcribes design proposals into structured JSON templates containing layer models, dimensions, colors, and content strings.
*   **To Chat Modules:** Generates friendly conversational reports, explaining recommendations and offering design variants.
*   **To Knowledge Graphs:** Saves successful visual compositions as customizable, modular templates for future reuse.
*   **To Workspace Panels:** Highlights modified layers with temporary focus outlines to direct user attention during collaborative adjustments.
